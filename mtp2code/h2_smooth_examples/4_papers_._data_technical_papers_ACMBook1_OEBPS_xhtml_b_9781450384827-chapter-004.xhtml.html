1: 4Design Principles of Event Mining Systems

2: We live in an era of data abundance

3: The availability of large volumes of multimodal data streams, ranging from videos, tweets, and other social media content to human activity, location, and environmental variables, can now be used to solve many critical societal problems

4: But as the emphasis on collecting data increases, there is a real need for a new generation of techniques, frameworks, and algorithms to assist researchers, analysts, and decision makers in extracting knowledge from such a variety of data

5: As the speed and scale of this data generation will increase even further in the future, we require new frameworks that support high-performance computing, processing techniques that fuse heterogeneous data and uncover hidden patterns and unknown correlations, scalable software tools, and useful visualizations that help analysts and decision makers understand the results better

6: Modeling complex, mysterious, and at least partly unknowable systems involves many complicated decisions such as determining a model selection strategy, defining a model structure, defining a criteria for model goodness, selecting data and the transformations applied to the data, tuning learning parameters, and so on

7: Most of these decisions involve a reliance on theoretical or empirical results, that is, expert domain knowledge, and cannot be learned by a system itself solely from available input data

8: Moreover, many spurious associations might arise from learned models, resulting in false scientific discoveries and false statistical inferences [Calude and Longo 2017]

9: A promising approach for modeling complex phenomenon is to adopt a human-in-the-loop approach in the data processing step

10: This integrates high-level expert knowledge into the modeling process by acquiring an expert s relevance judgments regarding a set of initial retrieval results

11: Despite the apparent benefits of such a perspective, frameworks that facilitate a seamless interaction between a domain expert and a traditional knowledge discovery process are not well studied

12: Figure 4.1 shows the human-in-the-loop in a modeling process rooted in event mining.

13: The ultimate goal of event mining is to extract knowledge from data and create explanatory models

14: Knowledge is either a verified hypothesis or machine-generated patterns that are novel, interesting, and human-understandable

15: We emphasize that knowledge is required to be representable in a linguistic form, that is understandable by humans and automatically usable by knowledge-based systems

16: The conversion of sub-symbolic patterns and trends in data to a symbolic human-understandable form is the most critical part of such interactive modeling

17: In this chapter, we review some of the most important characteristics of an event mining system, especially the need for a common event mining language that facilitates hypothesis formation, testing, and expansion

19: 4.1Data Fusion and Transformation

20: Data fusion is the process of combining data to estimate or predict the state of some aspect of the world

21: This process usually combines data from multiple sensors to achieve more specific inferences than could be achieved by using a single, independent sensor

22: While the concept of multisensor data fusion is not new, the emergence of new sensors and advanced processing techniques has made real-time fusion of data increasingly viable

23: Techniques to fuse data are drawn from a diverse set of disciplines, including multimedia computing and communications, digital signal processing, statistical estimation, control theory, and machine learning

24: Fused data from multiple sensors provide several advantages over data from a single sensor

25: First, if several identical sensors are used (e.g., identical accelerometer sensors attached to different body parts of a subject), combining the observations will result in an improved estimate of the subject s activities and movements

26: A second advantage involves using the relative placement or motion of multiple sensors to improve the observation process

27: A third advantage of using multiple sensors is observability improvement

28: For example, combining GPS readings with accelerometer measurements can lead to a better recognition and differentiation between activities such as cycling and driving

29: One area that benefits immensely from sensor fusion is the monitoring of complex mechanical equipment such as turbomachinery, helicopter gear trains, and industrial manufacturing equipment

30: For instance, in a motor vehicle monitoring system, sensor data can be obtained from accelerometers, temperature gauges, oil debris monitors, acoustic sensors, and infrared measurements

31: A predictive maintenance system would seek to combine these observations to identify precursors to failure, such as an abnormal gear wear, shaft misalignment, or bearing failure

32: The use of such monitoring is expected to reduce maintenance costs and improve safety and reliability

33: In addition to mechanical equipment, the human body is a complex system that can benefit from continuous monitoring, fusion of observations, and higher-level decision-making

34: Health professionals are trying to develop more personalized predictive techniques to prevent diseases by taking appropriate lifestyle and medical actions.

35: There are three basic approaches to fusing multisensor data:

36: low-level, or the direct fusion of raw sensor data;

37: feature-level, or converting raw sensor data to features (first-level abstraction), and fusion of the feature vectors usually by applying statistical estimation and machine learning techniques; and

38: decision-level, or processing the output of each classification result separately to achieve a high inference (second-level abstraction).

39: If the sensor data measures similar (commensurate) information from the same physical phenomena, such as two visual image sensors or two accelerometer sensors on different limbs, then the raw sensor data can be directly combined

40: Techniques for raw data fusion typically involve classic estimation methods, such as Kalman filtering

41: If the sensor data is non-commensurate (i.e., sensors observing the same physical phenomena with dissimilar information), then the data must be fused at the feature vector level

42: Feature-level fusion involves the extraction of representative features from sensor data

43: In this case, features are extracted from multiple sensor observations and combined into a single concatenated feature vector

44: Fusion of principal component analysis (PCA) and linear discriminant analysis (LDA) coefficients of face images and the fusion of face and hand modalities are examples of fusion at the feature level

45: Feature-level fusion has also been used in medical imaging where multiple image types are utilized together to take advantage of cross-information

46: This proves helpful in recognizing a change in the functional magnetic resonance imaging (fMRI) activation maps associated with a change in the brain structure [Calhoun and Adali 2009].

47: Decision-level fusion is the most appropriate fusion technique for non-commensurate data that is sampled at different time points

48: Mangai et al

49: [2010] defined three categories for decision-level fusion: abstract, rank, and measurement levels

50: In a nutshell, the majority voting [Kittler et al

51: 1998], weighted majority voting [Littlestone and Warmuth 1994], and naive-Bayes combination [Kuncheva 2004] methods belong to the abstract level; the class set reduction (including intersection of neighborhoods and union of neighborhoods) and the class set reordering methods (including highest rank method, Borda count method, and logistic regression method) [Ho et al

52: 1994] belong to the rank level; and the class-conscious methods [Lam and Suen 1995] and the class-indifferent methods [Kuncheva et al

53: 2001] belong to the measurement level.

54: In the literature [Dasarathy 1994], decision-level fusion is considered as the highest level of abstraction

55: As mentioned, it is a combination of classifiers to achieve better classification accuracy in pattern recognition problems

56: We can clearly see that decision-level fusion is indeed helpful in aggregating data from asynchronous sources of information

57: However, for longitudinal analysis of the behavior of a system, and based on discussions around event models in Chapter 2, we need an event-level fusion.

58: In various disciplines, information about an underlying phenomenon might be acquired from different types of sensors

59: Rarely does a single modality provide complete knowledge of the phenomenon of interest due to the rich characteristics and complexity of that phenomenon

60: Hence, sensor fusion is an inevitable pre-processing step in any event mining framework

61: However, a main challenge is how to fuse these modalities into a human-understandable abstraction signal that not only preserves the semantics of the underlying system but also facilitates modeling

62: Data from different sources usually results in different silos that do not communicate with one another and are indexed using data-centric approaches

63: In the current form, it is not possible to make sense of these diverse sources of data

64: One way to address this issue is by organizing all this data around meaningful events so that we can overcome heterogeneity and variety issues in the data and fuse non-commensurate observations in a meaningful way

65: Event models help us to understand the underlying heterogeneous real-life events in multimodal content and use them to better organize, retrieve, and discover knowledge in any possible way

66: Event models assign different properties to an event, and different observation sources help in our interpretation and modification of these event properties

67: For instance, an exercise event can have multiple properties such as heart rate, calorie consumption, speed, distance traveled, and number of floors climbed

68: Some of these properties can be recognized from motion sensors, and some can be recognized from physiological sensors

69: Figure 4.2 shows multiple data streams from various sources being stored in data silos and consumed by applications separately

70: However, using an effective event model helps fuse these data streams at the event level and creates multiple event streams

71: The properties of each event in the stream might be populated from disparate sensor measurements.

72: Figure 4.2Multiple data streams from various sources are usually stored in data silos and they are consumed by applications separately

73: Using an effective event model helps in fusing these data streams at the event level.

74: In summary, data transformation and event-level fusion provides an abstraction layer for the essential structural and functional properties of an event mining system, hiding the varieties of input data streams and complexities of low-level event extraction algorithms

75: An event is an abstract bundle that has meaningful attributes for specific applications, and these specified attributes play a key role in event mining

76: 4.2Extensibility and Reusability

77: Extensibility is a measure of the ability to extend a system with new functionalities with minimal or no effects on its internal structure and data flow

78: An event mining framework needs to provide an easy way to integrate new kinds of data streams, data processing operators, and visualization techniques

79: Different event mining applications may share similar conceptual and functional models or implementation details

80: The availability of reusable design and processing components may facilitate the rapid development of such applications by enabling reconfiguration and composition of available building blocks

81: The system needs to provide capabilities for specifying and mining patterns of events at different layers of abstraction that regulate the creation of higher level pattern instances according to pattern matching conditions on lower-level events

82: Each application has its own requirement regarding the definition of patterns and the relationships between events

83: High-level abstraction operators that are easily extensible and reusable are essential components of a well-designed event mining system

84: In summary, a highly modular tool with its main functional units well isolated in order to ease maintenance and extension of operations is the core principle of an event mining system

85: 4.3Interactive Process

86: As shown in Figure 4.1, the event mining process for explanatory modeling is interactive

87: It is a closed-loop system that involves extracting patterns, formulating, refining, and validating a hypothesis and extracting patterns again in the light of new knowledge

88: Hence, the event mining system should incorporate an iterative query processing capability that both allows the analyst to apply data-driven algorithms to discover interesting patterns and formulate hypotheses to investigate these patterns further for a deeper analysis

89: The hypothesis formulation step is very important in creating hypotheses that are not only based on current observations but also accumulated domain knowledge

90: This is an iterative process that allows the analyst to use the models as static knowledge to be presented as results as well as an active element of the process used to go deeper into data understanding.

91: Interestingly, even data-driven hypothesis formation may not be completely data driven

92: In most cases, the data streams that should be used in correlation and the other factors that should be considered come from some domain knowledge or prior experience

93: Therefore, this process is actually a knowledge-mediated event mining process for modeling

94: The ability to precisely articulate relationships among events becomes very important, which is where an expressive event language can play a huge role

95: Moreover, discovered models and extracted patterns have to be explicitly represented and stored to allow the progressive mining and reuse of extracted models with different data.

96: 4.4Human-centered Analysis

97: Traditionally, having a human-in-the-loop in AI leverages both human and machine intelligence to create machine learning models

98: Humans label data, and a machine learning algorithm learns to make decisions from this data

99: Humans also tune, test, and validate the model as well as adjust it in situations where an algorithm is not confident about a judgment or overly confident about an incorrect decision

100: In real-world data analysis problems, domain experts typically use visualization techniques from common software/machine learning tools to visualize model results

101: Often, the underlying computations are not transparent and comprehensive enough to provide the feedback needed to guide the model refinement process

102: So, by integrating machine learning algorithms with interactive visualization, visual analytics aims to provide visual platforms for analysts to interact directly with data and models

103: In a case study, Tam et al

104: [2017] illustrated that human-centric machine learning can produce better results than purely machine-centric methods

105: By combining visual analytics and machine learning, an analyst can steer the computation and interact with the model and data through an interactive visual interface.

106: Visual approaches alone are limited by data size and representation design, as well as how observant a user is

107: Therefore, automatic identification of patterns is important in enhancing the model building task, eliminating noise, and bringing interesting patterns into focus

108: Most often frequency is specified as the characterizing attribute of an identified pattern

109: This is, however, not necessarily the desired feature in all applications

110: Allowing a user to specify the characteristics of the interestingness of patterns can instead lead to the discovery of patterns that are important with respect to other factors

111: Furthermore, even constraining the event mining process by specifying the desired attributes can still lead to a huge number of resulting patterns

112: This enhances the benefit of introducing interactivity in the search for patterns

113: Adopting such a method eliminates the need to identify a large number of patterns first and then filter these patterns to retain the interesting ones

114: Interactivity allows the user to select and pursue only directions of interest during the model building process.

115: With increased computing power and speed, deep learning has made significant advancements in terms of efficiency and speed and has had a considerable impact on various long-running AI problems, including computer vision, speech recognition, and natural language understanding

116: As humans increasingly rely on artificial intelligence (AI) techniques, the interpretability of their decisions and the control over their internal processes are becoming a serious concern for sensitive domains such as precision medicine and bioinformatics

117: In domains where the costs of false positives and false negatives tend to be higher, it is important to leverage the knowledge of subject matter experts to the greatest extent and with the utmost precision

118: The cost of misclassification of a cat image versus a dog image is not high

119: However, the cost of misdiagnosing a deadly disease is irreparable

120: Girardi et al

121: [2016] emphasized the importance of a doctor-in-the-loop concept in knowledge discovery and proposed an interactive data mining process in biomedical research

122: In an effort to increase explainability in machine learning models, researchers have started to look inside black-box machine learning models and emphasize the importance of understanding why an algorithm acts a certain way

123: The Defense Advanced Research Projects Agency (DARPA) in the United States is launching an initiative called eXplainable artificial intelligence (XAI) to bring attention to the importance of understanding, trusting, and effectively managing AI models [Gunning 2017a].

124: One way to look inside a machine learning black box is to provide visualizations of what is happening in an algorithm because it opens up interesting challenges at the intersection of machine learning and visual analytics

125: In this direction, a number of conceptual frameworks have been proposed, such as iPCA [Jeong et al

126: 2009], which lets analysts visualize the results of PCA by using multiple coordinated views and a set of user interactions

127: iPCA provides four views, each representing a specific aspect of the input data either in data space or eigenspace, and is coordinated in such a way that any interaction with one view is immediately reflected in all the other views

128: ForceSPIRE [Endert et al

129: 2012] introduces the concept of semantic interaction, which leads to a design space for user interaction with textual information

130: Using the system, analysts can add textual annotations to different documents and include those annotations in the analysis process

131: To capture domain expert knowledge in the model building process, ForceSPIRE allows users to lay out documents visually such that the layout reflects meaningful notions of similarity and distance between the documents

132: 4.5Event Mining Architecture

133: The goal of reviewing design principles in an event mining framework is to propose a truly extensible, reusable, flexible, human-centered, event-driven architecture for explanatory modeling

134: This architecture should provide seamless connectivity between different components, fluid communication between different levels (batch processing and real-time analysis), and manage the recognition and indexing of different types of events and their properties

135: In various disciplines, not only does information need to be captured from different sensor modalities, which in return results in event-level fusion in the system, but also decisions should be made depending on more than one event

136: The event mining platform needs to support two separate processing environments: offline model building and online model deployment

137: The offline model building environment is similar to a training/test phase in machine learning, where a model is trained and evaluated on historical data

138: The model building phase in an event mining platform involves applying event mining algorithms on longitudinal event streams to extract significant patterns from data

139: This process is iterative, interactive, and involves humans in hypothesis formulation, evaluation, and refinement

140: The algorithms need to be extremely efficient in processing the large amounts of data collected over a period of time

141: The end result of this process is a set of rules (i.e., extracted patterns) that encode complex interactions between a set of events (e.g., the cause) that induce a target event (e.g., the effect)

142: For example, to understand the risk factors of an asthma attack in a patient, we might find significant patterns such as:

143: PM2.5-inc followed by asthma attack

144: (PM2.5-inc and temperature high) followed by asthma attack

145: These patterns can be encoded as rules and utilized in an online stream processing task for recommendation and feedback.

146: Online model deployment is similar to complex event processing environments, where streams of data flow into the system, and a set of predefined rules act as standing queries

147: If a rule is triggered, an alert will be generated

148: Any architecture for complex event processing should have the ability to import data from multiple sources, apply rules, and derive outbound actions

149: A rule engine can vary from a simple domain-specific language to a complex business rule management system

150: However, regardless of the complexity of the rules, they are predefined by an expert

151: An event mining framework aims to extract these rules in a model building environment by applying a combination of data-driven and hypothesis-driven event mining operators

152: Figure 4.3 shows the high-level architecture of an event mining platform

153: The main components of this platform are as follows:

154: Data sources

155: All the data available for analysis from different sources, such as environmental, traffic, wearable, mobile apps, and so on, come into the system in a structured, semi-structured, or unstructured format

156: The speed that the data arrive and the rate at which they are delivered varies according to the data source

157: At the collection point, data are collected directly or through data providers in real time or in batch mode.

158: Data ingestion

159: Although data ingestion is not necessarily an internal part of the event mining platform, it is the beginning of the data pipeline and the focus of data transport from its source to everywhere else

160: Some data sources have tables or databases that are frequently updated to reveal certain sensory data collected by different agencies

161: Data can be streamed in real time or ingested in batches

162: Real-time data are consumed by an online event processing component as soon as new data are available

163: When data are ingested in batches, data items are ingested in chunks at periodic intervals

164: With the availability of numerous data sources, data exist in different formats, which is a big challenge for applications

165: They have to ingest data at a reasonable speed and convert data to a correct format

166: Figure 4.3The high-level conceptual architecture of an event mining framework.

167: Event recognition and event management

168: Most of the data coming from sensors (physical, social, etc.) are in the form of data streams

169: The event mining platform should handle and aggregate multiple heterogeneous streams, synchronize them, and provide stream processing operations

170: To support effective and efficient model building, streaming data need to be managed at an event level

171: Primitive sensor data also need to be filtered, aggregated, and correlated to generate more semantically meaningful complex events with a rich predefined event schema

172: Different recognition techniques can be utilized to convert raw sensor data to events, ranging from simple time-series discretization to complex machine learning methods

173: Model building

174: Hypothesis formulation, hypothesis evaluation, and event mining are important subtasks in an event mining system

175: The end goal is to build an explanatory model

176: One way to formulate this model is by converting extracted patterns to explainable rules

177: These pattern are either verified hypotheses (generated by applying hypothesis formulation and evaluation) or machine-generated patterns that are novel, interesting, and human-understandable

178: In the model building component, different pattern mining and machine-learning techniques should be included to detect co-occurrences and causal explanations among different events

179: Human-in-the-loop for hypotheses formulation, hypotheses verification, and to verify hypotheses and examine investigation of machine-generated patterns is only possible with user-friendly dashboards and expressive visualizations

180: Model deployment

181: Deployment is a process in which a trained model is integrated into the environment for data-based decision-making

182: Many applications require continuous data monitoring and processing in a timely fashion as data flows from source to system

183: The key requirement here is early identification of a situation of interest and quick response generation

184: The model generated in the previous step can be considered as a set of causal explanation rules, and might be deployed in a similar setting as complex event processing (CEP) systems

185: CEP technology is addressing continuous data monitoring requirements through an infrastructure that consists of three main components: a set of event sources, a rule engine, and a set of event sinks or event listeners

186: The rule engine consists of a set of rules that act as continuous queries or pattern signatures that filter, aggregate, and correlate events as they flow into the system

187: Conventional CEP rules are manually defined by domain experts

188: In the event mining framework, the rule set is extracted by pattern mining and hypothesis verification

189: The rule set is configured once but might change later as pattern mining algorithms run in the background and more interesting patterns are discovered
